Code Log 5: Images

How images are used in HTML and CSS

Images can be used in both HTML and CSS, but their purpose and functionality are a bit different. The <img> tag is used in HTML to directly display an image as part of the page’s content. It’s ideal when the image itself is important — like a product photo, logo, or something users need to see. On the other hand, background-image in CSS is mostly decorative. It’s applied to an element’s background and doesn’t affect the main structure of the page. The <img> tag can use alt text, which helps with accessibility, while background images are not read by screen readers. In short, <img> is for content, and background-image is for design or layout styling.

Example of an HTML image:

<img src="cat.jpg" alt="Cute cat">

Example of a background image:

div {
  width: 300px;
  height: 200px;
  background-image: url('cat.jpg');
  background-size: cover;
  background-position: center;
}

Other CSS properties that affect background images

Besides background-image, there are several CSS properties that control how background images look and behave:

  • background-size: Controls how the image fits inside an element. For example, cover makes the image fill the area, while contain ensures the whole image is visible without being cropped.
  • background-repeat: Defines whether the image repeats across the background (values include no-repeat, repeat-x, and repeat-y).
  • background-position: Sets where the image appears, like center, top left, or bottom right.
div {
  background-image: url('pattern.png');
  background-repeat: no-repeat;
  background-size: contain;
  background-position: center;
}

Dimensions vs. resolution

Dimensions describe an image’s physical size in pixels (for example, 800×600), while resolution measures how detailed or sharp the image looks, usually in pixels per inch (PPI or DPI).

Extra Credit: Image File Formats

  • GIF: Great for simple animations or low-color graphics, but not ideal for detailed photos.
  • PNG: Keeps images crisp and supports transparency — perfect for logos or design graphics.
  • JPG/JPEG: Best for photos because it compresses file sizes, though some quality is lost.
  • WebP: A modern format that balances quality and file size, great for faster website loading.
  • SVG: Vector-based, so it scales perfectly without losing quality — ideal for icons or logos.

In short: JPEGs are great for photos, PNGs for transparent graphics, GIFs for animations, WebP for modern web optimization, and SVGs for infinitely scalable vector designs.
