Code Log 1: HTML Semantic Tags

What is the difference between non-semantic and semantic tags in HTML?

Semantic tags in HTML actually tell you what the content is. For example:

  • header
  • nav
  • article
  • section
  • footer

All of these above give meaning to the stuff inside them. They make your code easier to read, better for SEO, and more accessible for things like screen readers.

Non-semantic tags don't explain anything about the content.

and are the big ones, they're just generic containers that are mainly used for styling or grouping things together with CSS.

The simple way to think about it:

  • Semantic=has meaning.
  • Non-semantic=just boxes with no meaning.

What are two (or more) benefits of using semantic tags?

Two big benefits of using semantic tags are:

  • Better accessibility: screen readers and assistive tech can understand your page structure more easily.
  • SEO boost: search engines know what your content means, so your site can rank better.
