Code Log 3: CSS Media Queries

How does a CSS media query work, and why is it useful?

A CSS media query works by checking the conditions of a device (like screen size or orientation) and applying specific styles if those conditions are true. It basically tells the browser. "if this situation matches, use these styles." Media queries are super useful because they make websitews responsive, meaning they adjust layouts to look good on phones, tablets, laptops, or big destop screens without needing separate websites.

What "media features" are supported by media queries — such as "orientation" or "max-width" — and what do they do?

Four Media Features:

1.max-width

Applies styles only when the screen is at or below a set width. Super common for making mobile breakpoints.

Example:

@media (max-width: 768px) {
        body {
            font-size: 16px;
        }
    }

→ apply styles when the screen is tablet-sized or smaller.

2.min-width

The opposite of max-width. It applies when the screen is at or avobe a set width. Usually used for desktop layouts.

Example:

@media (min-width: 1024px) {
        body {
            font-size: 16px;
        }
    }

→ styles for larger screens like laptops and desktops.

3.orientation

Detects whether the screen is portait (taller than wide) or landscape (wider than tall). Handy for phones and tablets when people rotate their devices.

Example:

@media (orientation: landscape) {
        body {
            font-size: 16px;
        }
    }

→ apply styles when the device is sideways.

4. prefers-color-scheme

Matches a user's system setting for light mode or dark mode. This lets your site automatically adapt to their preference.

Example:

@media (prefers-color-scheme: dark) {
        body {
            background: #000;
            color: #fff;
        }
    }

→ switches the site to dark mode if the user prefers it.
